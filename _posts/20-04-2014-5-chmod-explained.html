chmod is a very common command that every linux user should learn about. It is a typical example for the simplicity of *nix systems. This article describes what is chmod and how does it work.

chmod stands for "change mode". It is used to change the access permission of a file. chmod basically needs two arguments, the permission and the filename. In linux there are three types of permissions for a file viz. read, write and execute. These permissions can be set for three levels of users separately- the owner(user who created the file), the group(users who shares same group with the owner), and the world. The three permissions(read, write and execute) are represented by octal numbers in linux. these are explained below

Octet number

Description

1

read

2

write

3

Read and write(1+2)

4

execute

7

read, write and execute(1+2+3)

  
  
 
 

Below is an example

$ chmod 743 ~/document.txt
Here, this is what the "743" means.

7 - read, write and execute permission for the file owner,
4 - read and write permission for users in the same group as of the file owner,
3 - read only permission for rest of the world.

You can change the permissions of all files in a folder by using the -R option.
Ex: $ chmod -R 743 ~/documents. Here documents is a folder.

There is another way of representing the permissions using r, w and x. Details can be found here. But I prefer using the octet method, as it's simpler.

PS: to make permission for a file same as another file, use $ chmod --reference=file1 file2
